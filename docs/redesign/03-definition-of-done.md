# Этап 0: Definition of Done Для Редизайна

## Общий DoD для каждой волны
Изменения считаются завершенными, когда выполнены все пункты ниже.

## 1. Функциональная корректность
1. Ключевые пользовательские сценарии экрана работают без регрессий.
2. Навигация между связанными экранами не ломает состояние без причины.
3. Нет критических ошибок в консоли при типичных действиях пользователя.

## 2. UX-качество
1. Экран соответствует правилам из `01-practical-ui-rules.md`.
2. Главный action экрана очевиден.
3. Сценарии `loading`, `empty`, `error` предусмотрены и визуально согласованы.
4. Интерфейс воспринимается компактным и минималистичным, без избыточных отступов.

## 3. Визуальная консистентность
1. Компоненты выглядят единообразно в рамках волны.
2. Стиль соответствует целевому направлению (shadcn-like).
3. Цвета, типографика, радиусы, отступы используют единые правила, а не точечные исключения.

## 4. Мобильная стабильность
1. Проверено на мобильных режимах (включая iOS Safari/PWA-сценарии).
2. Верхние/нижние safe-area не перекрывают контент.
3. Жесты скролла/перетаскивания не конфликтуют.

## 5. Инженерное качество
1. Код не создает дублирования без необходимости.
2. `typecheck` и сборка проходят.
3. Документация по экрану/волне обновлена вместе с кодом.

## 6. Приемка волны
1. Выполнен smoke-check по списку сценариев волны.
2. Продукт подтверждает соответствие:
   - “интуитивно понятно”,
   - “современно и продуманно визуально”.
3. Решение зафиксировано в docs перед переходом к следующей волне.
