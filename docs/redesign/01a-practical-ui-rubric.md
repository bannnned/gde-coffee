# Practical UI Rubric (Wave 1)

Короткий чеклист для быстрого ревью экрана перед merge.

## Как оценивать
Шкала по каждому пункту:
- `0` — не выполнено;
- `1` — частично выполнено;
- `2` — выполнено.

Итог:
- `17-20` — можно принимать;
- `13-16` — нужен дофикс в текущем PR;
- `<=12` — экран требует пересборки решения.

---

## 10 пунктов rubric

1. **Ясная цель экрана**  
Есть ли у экрана одна четкая задача, понятная с первого взгляда?

2. **Один главный action**  
Явно ли видно первичное действие и не конкурирует ли оно с остальными?

3. **Иерархия контента**  
Читается ли порядок `главное -> вторичное -> служебное` без усилий?

4. **Минимализм без потери смысла**  
Нет ли визуального мусора, лишних блоков и декоративных элементов без функции?

5. **Spacing и выравнивание**  
Отступы и выравнивания системные, без случайных исключений?

6. **Типографика и читаемость**  
Текст легко сканируется, контраст достаточный, роли текста стабильны?

7. **Состояния интерфейса**  
Обработаны ли `loading / empty / error` без скачков layout и неясных сообщений?

8. **Формы и тексты действий**  
Лейблы, ошибки и CTA конкретны и однозначно объясняют следующий шаг?

9. **Мобильный UX и iOS-safe-area**  
Ничего не уезжает под системные зоны, touch-цели удобны, жесты не конфликтуют?

10. **Консистентность с системой**  
Экран визуально и поведенчески соответствует `01-practical-ui-rules.md` и остальным экранам волны?
